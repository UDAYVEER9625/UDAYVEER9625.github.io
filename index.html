Pizza Ordering System

System Design · Full-stack Flow
End-to-End System Design UX Thinking
The Problem Design a complete ordering system that handles the full user + admin journey without breaking under edge cases
My Role Mapped every user journey state, designed the data model, built the frontend, and structured backend logic
Key Decisions I Made
  • Started with a flow diagram, not code — caught 3 edge cases before touching HTML
  • Separated admin and customer views early — cleaner system, cleaner thinking
  • Prioritized order-status visibility as the #1 user need
Outcome A fully working system with clean state management — more importantly, proof that starting with a map beats starting with code.

Student Impact Dock

System Design · Data Pipeline
Python pandas ReportLab
The Problem Schools lacked a centralized system to track and evaluate student performance holistically — beyond just exam scores
My Role Defined the scoring methodology, designed the data model, and built the full reporting pipeline end-to-end
Key Decisions I Made
  • Weighted scoring (40% academics / 30% activities / 30% community) to reflect multi-dimensional growth

05

SmartAgro – Farm Management System

System Design · Offline Architecture
Java File I/O CSV Storage
The Problem Small farms needed a simple way to manage profiles, crops, and tasks — without relying on internet connectivity
My Role Designed the system architecture, data layer, and task scheduling workflow from scratch
Key Decisions I Made
  • CSV-based persistence for full offline capability — no database, no server, no internet required
  • Modular feature separation: farmer profiles, crop management, and task scheduling as independent modules
  • Productivity summaries generated by aggregating task completion data across time periods
Outcome A reliable, offline-first farm management tool that proves simple architectures can solve real-world problems effectively.
